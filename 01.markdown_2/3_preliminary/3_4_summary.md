# 本章のまとめ

本章では、システム設計に先立つ予備調査の結果を報告した。

既存システムの分析から、現行アプリケーションには精度、対応品種、リアルタイム性において改善の余地があることが明らかになった。ユーザーインタビューからは、迷子対策、健康管理、多頭識別、リアルタイム処理という4つの主要ニーズが抽出された。プロトタイプテストでは、ベースラインモデルの性能と改善すべき課題（類似品種の混同、遮蔽への対応、姿勢変化への頑健性）が特定された。

これらの知見は、次章で述べる提案システムの設計において、以下のように反映される。

1. ステイクホルダー要求として、ユーザーインタビューで抽出されたニーズを明文化する
2. 技術的要求として、既存システムを上回る精度（mAP@0.5で90%以上）を設定する
3. アーキテクチャ設計において、プロトタイプで特定された課題に対応する機構を組み込む

次章では、これらの予備調査結果を踏まえた提案手法の詳細を述べる。
